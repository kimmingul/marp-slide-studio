# Font Bundle

By default, all themes import fonts via CDN (`jsdelivr`, `Google Fonts`). That's zero-setup but requires internet at render time, and the rendered PDF/PPTX fonts depend on the CDN being reachable from Chrome's rendering environment.

For **offline use**, **team reproducibility**, or **air-gapped environments**, bundle fonts locally.

## Quick start

```bash
# From the plugin root
bash scripts/fetch-fonts.sh
```

This downloads:
- **Pretendard Variable** (SIL OFL) → `assets/fonts/pretendard/`
- **Noto Serif KR** 300/500/600/700 weights (SIL OFL) → `assets/fonts/noto-serif-kr/`
- **JetBrains Mono** 400/500/700 weights (SIL OFL) → `assets/fonts/jetbrains-mono/`

Total: ~8 MB.

## Switching a theme to offline

After fonts are downloaded, swap the top `@import` block of any theme CSS with the offline bundle:

```css
/* BEFORE — CDN */
@import url('https://cdn.jsdelivr.net/gh/orioncactus/pretendard@v1.3.9/...');
@import url('https://fonts.googleapis.com/css2?family=Noto+Serif+KR...');

/* AFTER — offline */
@import url('../../fonts/offline.css');
```

`assets/fonts/offline.css` (generated by `fetch-fonts.sh`) provides `@font-face` declarations for all bundled families.

## Why this isn't default

CDN is simpler for new users. Offline is for teams that ship decks through environments without internet (conference halls with bad wifi, air-gapped review sessions). Pick per-project.

## Swap all themes at once

```bash
# Convert every theme in this plugin to offline mode
for css in assets/design-systems/*/*.marp.css; do
  # Remove CDN @import lines
  sed -i.bak '/^@import url.*jsdelivr\|@import url.*fonts.googleapis/d' "$css"
  # Prepend offline bundle
  { echo "@import url('../../fonts/offline.css');"; cat "$css"; } > "$css.new"
  mv "$css.new" "$css"
  rm "$css.bak"
done
```

(Don't run this blindly — inspect the diff first.)
